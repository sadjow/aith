# Releasing aith

`aith` ships through these channels:

- GitHub Releases with prebuilt binaries and checksums, generated by `cargo-dist`.
- Shell and PowerShell installers attached to each GitHub Release.
- Homebrew formula publishing to `sadjow/homebrew-tap`.
- npm package publishing as `@sadjow/aith`.
- crates.io through `cargo publish`.
- Nix flakes through `nix run github:sadjow/aith`.

## One-time setup

The release workflow is committed in `.github/workflows/release.yml`.

The Homebrew tap repository exists at:

```sh
gh repo view sadjow/homebrew-tap
```

To let release CI publish the generated Homebrew formula, create a fine-grained
GitHub token with contents read/write access to `sadjow/homebrew-tap`, then set:

```sh
gh secret set HOMEBREW_TAP_TOKEN --repo sadjow/aith
```

To let release CI publish to crates.io, create a crates.io API token, then set:

```sh
gh secret set CARGO_REGISTRY_TOKEN --repo sadjow/aith
```

To let release CI publish the npm installer package, create an npm automation
token for the `@sadjow` scope, then set:

```sh
gh secret set NPM_TOKEN --repo sadjow/aith
```

Do not store a broad personal GitHub token in repository secrets.

`HOMEBREW_TAP_TOKEN` and `NPM_TOKEN` are required before pushing a release tag
because the release workflow includes Homebrew and scoped npm publish jobs.

`CARGO_REGISTRY_TOKEN` is required for crates.io publishing. Without it, the
GitHub Release can still complete, but the `Publish crate` workflow will skip
crates.io.

## Preflight

Run the full local release preflight:

```sh
devenv shell ci
nix flake check
```

Useful focused checks:

```sh
devenv shell dist generate --check
devenv shell dist plan
devenv shell publish-dry-run
```

## Release

1. Update `version` in `Cargo.toml`.
2. Update `CHANGELOG.md`.
3. Commit the release changes.
4. Create and push a semver tag:

```sh
git tag -a v0.1.0 -m "v0.1.0"
git push origin main v0.1.0
```

The `Release` workflow builds platform artifacts, uploads checksums, creates the
GitHub Release, and publishes the Homebrew formula when `HOMEBREW_TAP_TOKEN` is
configured.

The `Publish crate` workflow runs when the GitHub Release is published. It
publishes to crates.io when `CARGO_REGISTRY_TOKEN` is configured; otherwise it
exits cleanly after explaining that publishing was skipped.

If the npm publish job needs to be rerun for an existing release, use the manual
workflow:

```sh
gh workflow run "Publish npm package" --ref main -f tag=v0.1.0
```

## Install Commands

GitHub installer:

```sh
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/sadjow/aith/releases/download/v0.1.0/aith-installer.sh | sh
```

PowerShell installer:

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://github.com/sadjow/aith/releases/download/v0.1.0/aith-installer.ps1 | iex"
```

Homebrew:

```sh
brew install sadjow/tap/aith
```

npm:

```sh
npm install @sadjow/aith@0.1.0
```

crates.io:

```sh
cargo install aith --locked
```

Nix:

```sh
nix run github:sadjow/aith
```
